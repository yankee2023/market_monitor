# Design From Spec Template

以下を GitHub Copilot へそのまま入力し、仕様書から設計書を生成する。

---

あなたは C# / WPF アプリケーションの設計者です。

次に与える仕様書だけを入力として、実装可能な設計書を Markdown で出力してください。

制約:
- 仕様書に存在しない機能を追加しないこと
- 要求 ID をそのまま保持すること
- 機能は縦割り Feature 単位で分割すること
- Shared 配下には複数 Feature で共有が確定した責務のみ置くこと
- Composition は依存関係の組み立てだけに限定すること
- ViewModel は Feature 入口インターフェースのみに依存させること
- 各 Feature ごとに目的、責務、依存先、公開インターフェース、DTO、例外契約、テスト観点を出力すること
- 仕様書と設計書の整合性が確認できるよう、要求トレーサビリティ表を含めること

出力フォーマット:
1. 文書目的
2. 設計方針
3. システム構成
4. Feature 設計
5. 画面設計
6. Composition 設計
7. テスト設計
8. 整合維持ルール
9. 実装マッピング

品質条件:
- 仕様上の用語と設計上の用語を一致させること
- public API 名は実装へそのまま落とせる粒度にすること
- クラス責務は 1 クラス 1 役割に収まるようにすること

入力仕様書:

<<<SPECIFICATION
[ここに SPECIFICATION.md の全文を貼る]
SPECIFICATION

---

出力後に必ず自己点検してください。

自己点検項目:
- FR-01 から FR-10 まで欠落がないか
- 各 FR に少なくとも 1 つの設計要素が割り当てられているか
- Shared と Features の責務境界が逆転していないか
- 仕様にない機能や画面項目を追加していないか
