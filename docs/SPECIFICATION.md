# Tokyo Market Technical 仕様書

## 1. 文書目的
本書は、Tokyo Market Technical を東証銘柄のテクニカル分析専用アプリケーションとして再定義し、仕様書から設計書、設計書から実装を自動生成できる粒度で要求を固定するための仕様書である。

本書の目的は次の 4 点とする。

- 利用者視点の要求を、曖昧さのない機能仕様として定義する。
- 仕様書から設計書を生成するために、責務境界、入出力、制約、異常系を固定する。
- 設計書から実装を生成するために、型、メソッド、画面項目、永続化項目、テスト観点を固定する。
- 仕様書、設計書、実装の三者の整合性を継続的に維持するための更新規約を定義する。

本書では各要求に一意な要求 ID を付与し、設計要素と実装要素に追跡可能な形で対応づける。

---

## 2. システム概要

### 2.1 システム名
- Tokyo Market Technical

### 2.2 システムの目的
- 個人投資家が、東証プライム、スタンダード、グロース銘柄の現在値、会社名、ローソク足、保存済み履歴を単一画面で確認できるようにする。

### 2.3 対象ユーザー
- 日本株を中心に監視する個人投資家
- 東証プライム、スタンダード、グロース銘柄を素早く確認したいユーザー

### 2.4 動作環境
- OS: Windows
- アプリ種別: WPF デスクトップアプリケーション
- ネットワーク: Yahoo Finance、Stooq、JPX 一覧取得のためのインターネット接続を前提とする
- 保存先: ローカルファイルシステム上の SQLite データベースおよびログファイル

### 2.5 システム境界
- 本システムが担当する範囲
  - 東証上場銘柄入力の解釈
  - 日本株の現在値取得
  - 日本株のローソク足取得
  - チャート指標の計算と表示
  - 価格到達通知
  - セクター比較表示
  - 履歴保存と履歴表示
  - ステータスおよびログ出力
- 本システムが担当しない範囲
  - 為替レートの取得と表示
  - 米国株など東証以外の市場データ取得
  - 売買注文の発注
  - 資産管理
  - ポートフォリオ分析
  - ユーザー認証

## 3. 用語定義
| 用語 | 定義 |
| --- | --- |
| スナップショット | ある時点における日本株の現在値と更新時刻をまとめた現在値データ |
| 日本株 | 東証銘柄を表す .T 付きシンボルを持つ銘柄 |
| 東証対象銘柄 | JPX 上場銘柄一覧において市場区分が プライム、スタンダード、グロースのいずれかに該当する銘柄 |
| ローソク足 | 始値、高値、安値、終値から構成される時系列価格データ |
| 分析ライン | ローソク足から自動生成、または利用者が 2 点指定で追加する直線 |
| 日足 | 1 日単位の OHLC データ |
| 週足 | 1 週単位の OHLC データ |
| 表示期間 | 1か月、3か月、6か月、1年のいずれかの期間条件 |
| 会社名表示 | 銘柄コードまたは銘柄名から解決された東証対象銘柄名の画面表示 |
| 共通部 | 機能横断で再利用される基盤機能群 |
| 機能固有部 | 単一機能に閉じたモデル、サービス、ViewModel 群 |

## 4. 業務要件

### 4.1 利用シナリオ
- ユーザーは銘柄名または銘柄コードを入力し、現在値を更新できること。
- ユーザーは銘柄名または銘柄コードを入力し、分析表示を実行できること。
- ユーザーは日足または週足のローソク足を確認できること。
- ユーザーはチャート指標の表示対象を選択できること。
- ユーザーは MACD 系テクニカル指標を確認できること。
- ユーザーは出来高を確認できること。
- ユーザーは RSI を確認できること。
- ユーザーは指定価格到達時に Windows デスクトップ通知を受け取れること。
- ユーザーは同一セクター銘柄の比較表示を確認できること。
- ユーザーは銘柄サマリーおよび同業比較で市場区分を確認できること。
- ユーザーは現在値更新のたびに保存される価格履歴を内部保持し、将来機能で利用できること。
- ユーザーはアプリ起動直後に、日本株の既定表示状態から操作を開始できること。

### 4.2 業務ルール
- 既定の銘柄入力値は 7203 とする。
- 4 桁の数字入力は東証銘柄コードとみなし .T を補完する。
- 東証プライム、スタンダード、グロース銘柄名の入力は JPX 上場銘柄一覧に基づいて解決する。
- サポート対象の市場区分は設定ファイルで切り替え可能とする。
- 現在値取得は Yahoo Finance を優先し、必要時のみ Stooq にフォールバックする。
- ローソク足取得は Yahoo Finance を優先し、必要時のみ Stooq にフォールバックする。
- ローソク足の色は、陽線を赤、陰線を青とし、ひげ色は実体色と同じにする。
- 仕様変更時は、仕様書、設計書、実装を同一の要求 ID 単位で同時更新する。

### 5.1 機能一覧

| --- | --- | --- |
| FR-01 | 銘柄入力解決 | 入力文字列を東証監視対象シンボルへ正規化する |
| FR-02 | 現在値取得 | 日本株の現在値を取得する |
| FR-03 | 銘柄分析表示 | ユーザー操作で対象銘柄の分析表示を実行する |
| FR-05 | 履歴保存 | 取得したスナップショットを SQLite に保存する |

| FR-08A | チャート指標表示切替 | チャート上の各指標の表示対象を切り替える |
| FR-08B | テクニカル指標自動計算 | MACD 系指標を自動計算して表示する |

### 5.2 FR-01 銘柄入力解決
#### 5.2.1 目的
- ユーザーが入力した銘柄名、銘柄コード、シンボルを東証対象銘柄の内部処理用シンボルへ一意に解決する。

#### 5.2.2 入力
- 入力文字列: 0 文字以上の文字列

#### 5.2.3 出力
- 正規化済みシンボル文字列

#### 5.2.4 正常系仕様
- 空文字または空白入力の場合、7203.T を返すこと。
- 4 桁数字の場合、末尾に .T を付加して返すこと。
- .T 付き入力は大文字化して返すこと。
- 既知の別名入力は定義済みエイリアスへ解決すること。
- 東証プライム、スタンダード、グロース銘柄名入力は JPX 一覧を参照して解決すること。

#### 5.2.5 異常系仕様
- JPX 一覧取得に失敗しても、入力解決処理全体を異常終了させてはならない。
- 東証対象銘柄として解決できない場合、利用者向けの入力エラーメッセージを返却可能な例外へ変換すること。

#### 5.2.6 受入条件
- トヨタ を入力した場合、7203.T として扱われること。
- 7203 を入力した場合、7203.T として扱われること。
- 東証対象銘柄名が一意一致した場合、該当 .T シンボルに解決されること。
- IBM を入力した場合、入力エラーメッセージが返ること。

### 5.3 FR-02 現在値取得
#### 5.3.1 目的
- 指定銘柄の現在値を取得し、画面表示用スナップショットを生成する。

#### 5.3.2 入力
- 正規化前の銘柄入力文字列
- キャンセルトークン

#### 5.3.3 出力
- スナップショット
  - Symbol
  - CompanyName
  - StockPrice
  - StockUpdatedAt

#### 5.3.4 正常系仕様
- 日本株の株価は Yahoo Finance を優先し、必要時のみ Stooq を利用すること。
- 取得成功時は現在時刻を更新時刻に設定すること。

#### 5.3.5 異常系仕様
- レート制限時は、利用可能なキャッシュがあればキャッシュ値を返すこと。
- Yahoo Finance が利用できない場合は Stooq を利用すること。
- 最終的に取得できない場合、利用者向けの失敗メッセージを返却可能な例外へ変換すること。

#### 5.3.6 受入条件
- 成功時に StockPrice が 0 より大きいこと。
- Symbol が .T で終わること。

### 5.4 FR-03 手動更新
#### 5.4.1 目的
- ユーザー操作により現在値取得、履歴保存、分析表示更新を一括実行する。

#### 5.4.2 トリガー
- 手動更新ボタン押下
- 表示ボタン押下
- 銘柄入力欄で Enter 押下

#### 5.4.3 処理仕様
- ステータスを 分析データ取得中... に変更すること。
- 現在値取得処理を呼び出すこと。
- 取得結果を画面へ反映すること。
- 履歴保存と履歴再読込を実行すること。
- ローソク足を再取得して表示を更新すること。
- 完了時は表示完了メッセージをステータスへ反映すること。

### 5.5 FR-04 自動更新
#### 5.5.1 状態
- 本機能は廃止し、本アプリケーションではサポートしない。

### 5.6 FR-05 履歴保存
#### 5.6.1 目的
- 取得したスナップショットをローカル DB に保存する。

#### 5.6.2 保存項目
- symbol
- stock_price
- recorded_at

#### 5.6.3 正常系仕様
- 保存前に price_history テーブルが存在しない場合は作成すること。
- 旧スキーマに exchange_rate 列が存在する場合は、新スキーマへ自動移行すること。
- 保存時刻はスナップショットの株価更新時刻を使用すること。

### 5.7 FR-06 履歴読込
#### 5.7.1 目的
- 指定銘柄の直近履歴を内部処理用に読み込む。

#### 5.7.2 入力
- シンボル
- 取得件数上限

#### 5.7.3 正常系仕様
- 既定取得件数は 20 件とする。
- 直近履歴は recorded_at 昇順で読み出すこと。
- 履歴読込結果は現在値更新後の整合確認や将来拡張で利用できること。

### 5.8 FR-07 日本株ローソク足表示
#### 5.8.1 目的
- 東証対象銘柄選択時に、ローソク足領域へチャートを表示する。

#### 5.8.2 入力
- シンボル
- 足種別
- 最大取得件数

#### 5.8.3 正常系仕様
- .T 付きシンボルのみを監視対象として扱うこと。
- Yahoo Finance を優先して OHLC を取得すること。
- Yahoo Finance が利用できない場合のみ Stooq を利用すること。
- 描画データは始値、高値、安値、終値を元に正規化すること。
- 陽線は赤、陰線は青、ひげは実体と同色にすること。
- ローソク足は縦軸に株価、横軸に日付を表示すること。
- チャート指標として 5 期間線、25 期間線、75 期間線を描画できること。
- 将来の出来高や MACD 追加を見据え、指標ごとに表示対象を管理できること。

### 5.9B FR-08B テクニカル指標自動計算
#### 5.9B.1 目的
- ユーザーが価格チャートとあわせて MACD 系指標を確認できるようにする。

#### 5.9B.2 対象指標
- 出来高
- MACD
- シグナル
- RSI

#### 5.9B.3 正常系仕様
- MACD は 12 期間 EMA と 26 期間 EMA の差分から算出すること。
- シグナルは MACD の 9 期間 EMA とすること。
- RSI は 14 期間を用いて算出すること。
- 出来高、MACD 系指標、RSI は下段指標パネルへ表示すること。
- MACD とシグナルは同一チェックボックスで表示制御すること。
- 下段指標パネルは指標ごとに分離して表示すること。
- 下段指標パネルは指標ごとに折りたたみできること。
- 下段指標パネルの表示高さは指標ごとに個別調整できること。

### 5.10 FR-13 市場区分表示と設定
#### 5.10.1 目的
- ユーザーが分析対象銘柄と同業比較銘柄の市場区分を確認できるようにする。
- アプリ再ビルドなしで対象市場区分を切り替えられるようにする。

#### 5.10.2 正常系仕様
- 補助ペインの銘柄サマリーに市場区分を表示すること。
- 同業比較カードに各銘柄の市場区分を表示すること。
- 市場区分設定ファイルは Prime、Standard、Growth の列挙値で指定できること。
- 設定ファイルが有効な場合、JPX 一覧の市場区分フィルタは設定値に従うこと。

#### 5.10.3 異常系仕様
- 設定ファイルが存在しない、または不正な場合は既定の Prime、Standard、Growth を使用すること。

#### 5.10.4 受入条件
- 7203.T 表示時に補助ペインで市場区分が表示されること。
- 同業比較カードごとに市場区分が表示されること。
- 設定ファイルから Standard のみを指定した場合、対象市場フィルタが Standard のみに切り替わること。

### 5.9 FR-08 ローソク足条件切替
#### 5.9.1 目的
- ユーザーが足種別と表示期間を切り替えられるようにする。

#### 5.9.2 切替条件
- 足種別: 日足、週足
- 表示期間: 1か月、3か月、6か月、1年

#### 5.9.3 正常系仕様
- 日足を既定値とすること。
- 表示期間は 1か月を既定値とすること。
- 条件変更時は、選択状態を ViewModel 上の真偽値へ反映すること。
- 条件変更時は、選択状態に応じた強調表示をボタンへ反映すること。

### 5.9A FR-08A チャート指標表示切替
#### 5.9A.1 目的
- ユーザーがチャート上の表示対象指標を切り替えられるようにする。

#### 5.9A.2 対象線種
- 5 期間線
- 25 期間線
- 75 期間線

#### 5.9A.3 正常系仕様
- 初期状態では MA5、MA25、MA75 を表示すること。
- 各指標は個別に表示/非表示を切り替えられること。
- 表示切替はチャート再取得を伴わず、保持済み描画データで反映すること。
- 非表示の指標は凡例にも表示しないこと。
- MA5 は黄色（#F59E0B）、MA25 は緑（#10B981）、MA75 は水色（#0EA5E9）の実線で描画すること。

### 5.9C FR-08C 分析ライン描画
#### 5.9C.1 目的
- ユーザーがローソク足チャート上へ任意本数の分析ラインを重ねて裁量分析できるようにする。

#### 5.9C.2 入力
- 表示対象ローソク足一覧
- 自動生成線選択ウィンドウでのチェック操作
- 手動描画線種選択ウィンドウでの種別選択
- 手動で線を描く操作
- 手動追加時のローソク足チャート上 2 点クリック
- 既存ライン選択
- ドラッグ移動操作
- 選択中の線を削除する操作
- すべての線を消去する操作

#### 5.9C.3 正常系仕様
- チャート表示時にトレンドライン、支持線、抵抗線をローソク足から自動生成すること。
- 自動生成結果は既存ラインへ追加する操作を提供すること。
- 「自動生成を追加」操作では別ウィンドウを開き、候補線をチェック選択して追加できること。
- 「手動で線を描く」操作では別ウィンドウを開き、トレンドライン、支持線、抵抗線から描画する線種を選択できること。
- 分析ライン描画モード開始後、チャート上の最初のクリックを始点、2 回目のクリックを終点として直線を追加すること。
- 分析ラインは複数本保持できること。
- 追加した分析ラインはローソク足や移動平均線に重ねて表示すること。
- 既存の分析ラインをクリック選択できること。
- 選択中の分析ラインはドラッグ移動できること。
- 選択中の分析ラインを個別削除できること。
- 画面上で各線の名称、色、意味を確認できること。
- 手動描画は「開始ボタン → 線種選択 → 2 回クリック（始点/終点）」の 3 ステップを画面内ガイドで確認でき、完了後は必要に応じてドラッグ調整できること。
- 表示期間の切替やチャート再読込後も、保持済み分析ラインを現在のチャート幅へ再配置して表示すること。
- 分析ラインは銘柄、足種別、表示期間ごとに保存し、同一条件の再表示時に復元すること。
- 保存済みラインが存在しない場合は自動生成ラインを初期値として採用すること。
- 最後に追加した分析ラインを 1 本ずつ削除できること。
- 保持中の分析ラインを一括で全消去できること。
- 自動生成アルゴリズムの概要（トレンド方向判定、ピボット点探索、クラスタリング基準）を画面上の折りたたみ式パネルで確認できること。

#### 5.9C.4 異常系仕様
- 始点と終点が極端に近い場合は分析ラインを追加しないこと。
- 分析ライン描画モードが無効な場合、チャートクリックを分析ライン追加として扱わないこと。
- 銘柄が切り替わった場合、前銘柄の分析ラインは引き継がないこと。
- ドラッグ移動時は分析ラインがチャート表示領域外へ完全に逸脱しないこと。

#### 5.9C.5 受入条件
- チャート初回表示時にトレンドライン、支持線、抵抗線が自動表示されること。
- 自動生成追加を実行した場合、既存ラインを残したまま不足分だけ追加されること。
- 「自動生成を追加」押下時に候補選択ウィンドウが表示されること。
- 「手動で線を描く」押下時に線種選択ウィンドウが表示されること。
- 線種選択後にチャート上を 2 回クリックすると、分析ラインが 1 本追加されること。
- 支持線または抵抗線を選択して追加した場合、種別に応じた色や線種で表示されること。
- トレンドライン（紫系）、支持線（青緑系）、抵抗線（赤系）が同時表示時でも視覚的に判別できること。
- MA75 は水色（#0EA5E9）の実線で表示されること。
- 自動生成アルゴリズムの説明を画面の折りたたみ式パネルで確認できること。

- アプリ再起動後に同じ銘柄、足種別、表示期間を開くと保存済み分析ラインが復元されること。
- 各線の意味を画面の凡例から確認できること。
- 手動描画ガイドに従って初見ユーザーが 3 ステップ以内で 1 本以上の線を追加できること。
- 「選択線を削除」で選択中の 1 本だけが消えること。

## 10. 非機能要件

### 10.1 保守性
- 共通部と機能固有部を分離した縦割り構造を採用すること。
- ViewModel は機能入口インターフェースのみに依存すること。
- UI はコードビハインドへ業務ロジックを持たないこと。
- 仕様書と設計書、設計書と実装の差分は同一変更内で解消すること。

### 10.2 拡張性
- 新機能は Shared 配下へ直接追加せず、まず Features 配下の独立機能として追加すること。
- 共通化が確定した責務のみ Shared 配下へ昇格させること。

### 10.3 テスト性
- 主要機能入口はインターフェースを持つこと。
- 外部 API 呼び出し、DB アクセス、時刻依存処理は分離可能であること。
- 要求 ID ごとに受入条件をテストへ追跡できること。

### 10.4 ローカライズ
- 利用者向けメッセージは日本語で表示すること。
- コメント、公開 API コメント、テスト説明は日本語で記述すること。

### 10.5 品質ゲート
- 実装、テスト、XAML、プロジェクト設定を含むビルド結果は、常にエラー 0 件かつ警告 0 件を維持すること。
- Roslyn Analyzer とコードスタイル診断は通常ビルドで有効化し、警告は失敗として扱うこと。
- Information レベル診断も新規発生を許容せず、修正または抑止理由を明示したうえで管理すること。
- 品質ゲートを満たさない変更は未完了とみなし、仕様・設計・実装の正本へ反映しないこと。

### 11.4 設計インターフェース生成規則

- 画面更新に必要な DTO は Feature/Models 配下に置くこと。

## 14. 実装受入チェックリスト
- 既定起動時に 7203 を入力状態として表示すること。
- 入力欄に IBM を指定した場合、東証銘柄を求めるエラーメッセージが表示されること。
- 4 桁コード入力で .T が補完されること。
- 東証銘柄名が一意に解決されること。
- 手動更新で現在値、履歴、ローソク足が一括更新されること。
- 履歴 DB に symbol, stock_price, recorded_at が保存されること。
- 旧 DB に exchange_rate 列があっても履歴保存と読込が継続できること。
- チャート領域に縦軸の株価ラベルと横軸の日付ラベルが表示されること。
- 横軸ラベルは可読性を優先して間引き表示されること。
- チャート上でマウスオーバーした足に対して、日付、始値、終値、高値、安値が表示されること。
- ローソク足の陽線が赤、陰線が青、ひげが同色であること。
- チャート指標の表示切替で、選択された MA5、MA25、MA75 だけが表示されること。
- 非表示の指標が凡例から除外されること。
- MACD とシグナルが同一チェックボックスで表示制御されること。
- 出来高バーが下段パネルへ表示されること。
- MACD とシグナルが下段指標パネルへ表示されること。
- RSI が下段指標パネルへ表示されること。
- 指定価格を跨いだタイミングで Windows 通知が発生すること。
- 業種名と同一セクター比較対象が右ペインへ表示されること。
- ログファイルが日次ローテーションされること。

## 15. 今後の拡張時の追記規則
- 新機能を追加する場合は、まず本書へ FR を追加すること。
- 次に 11 章の規約に従い設計書へ Feature 単位の責務を起こすこと。
- 最後に 12 章の規約に従い実装へ落とし込むこと。
- 実装追加後は 13 章のトレーサビリティ表を更新すること。
- 仕様変更時は、仕様書差分に対応する設計書差分と実装差分が同一コミットまたは同一変更セットに存在すること。

以上により、本書を起点として 仕様書 → 設計書 → 実装 の順に生成可能な粒度を維持する。
